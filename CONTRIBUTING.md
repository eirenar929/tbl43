## 🚀 Как внести вклад

### 1. Форкните репозиторий

Нажмите кнопку **Fork** в правом верхнем углу страницы репозитория.

### 2. Клонируйте форк

```bash
git clone https://github.com/your-username/table-for-three-v2.git
cd table-for-three-v2
```

### 3. Создайте ветку

```bash
git checkout -b feature/your-feature-name
```

Используйте префиксы:
- `feature/` — новая функциональность
- `fix/` — исправление ошибок
- `docs/` — изменения документации
- `refactor/` — рефакторинг кода
- `test/` — добавление тестов

### 4. Внесите изменения

Следуйте стандартам кода (см. ниже).

### 5. Закоммитьте изменения

```bash
git commit -m "feat: добавить новую функцию"
```

Используйте [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` — новая функция
- `fix:` — исправление ошибки
- `docs:` — документация
- `style:` — форматирование
- `refactor:` — рефакторинг
- `test:` — тесты
- `chore:` — вспомогательные изменения

### 6. Отправьте в форк

```bash
git push origin feature/your-feature-name
```

### 7. Создайте Pull Request

Перейдите на страницу вашего форка и нажмите **New Pull Request**.

## 📝 Стандарты кода

### Python

- Следуйте [PEP 8](https://pep8.org/)
- Используйте type hints
- Максимальная длина строки — 100 символов
- Используйте docstrings для функций и классов

Пример:
```python
def process_message(message: str, timeout: float = 10.0) -> dict:
    """
    Обрабатывает входящее сообщение и возвращает ответ.
    
    Args:
        message: Текст сообщения от пользователя
        timeout: Таймаут ожидания ответа в секундах
    
    Returns:
        Словарь с результатом обработки
    """
    pass
```

### Асинхронный код

- Используйте `async/await` для I/O операций
- Избегайте блокирующих вызовов в асинхронных функциях
- Используйте `asyncio.gather()` для параллельного выполнения

### Логирование

- Используйте модуль `logging`
- Уровни: DEBUG, INFO, WARNING, ERROR, CRITICAL
- Не логируйте чувствительные данные (API ключи, токены)

## 🧪 Тестирование

### Установка зависимостей для тестов

```bash
pip install pytest pytest-asyncio httpx websockets
```

### Запуск тестов

```bash
# Все тесты
pytest tests/ -v

# Только unit-тесты
pytest tests/ -v -k "not websocket"

# С покрытием
pytest tests/ --cov=. --cov-report=html
```

### Добавление тестов

Новые тесты должны:
- Иметь понятные имена
- Проверять один сценарий
- Быть независимыми друг от друга
- Использовать фикстуры для повторяющегося кода

Пример:
```python
@pytest.mark.asyncio
async def test_response_buffer_timeout():
    """Проверяет таймаут при ожидании ответа."""
    buffer = ResponseBuffer(timeout=0.1)
    result = await buffer.wait_all()
    assert result == {}
```

## 🔀 Процесс Pull Request

### Требования к PR

1. **Описание**: Подробно опишите изменения
2. **Тесты**: Добавьте тесты для новой функциональности
3. **Документация**: Обновите README.md при необходимости
4. **Changelog**: Добавьте запись в CHANGELOG.md

### Шаблон описания PR

```markdown
## Описание
Краткое описание изменений.

## Тип изменений
- [ ] Новая функция
- [ ] Исправление ошибки
- [ ] Рефакторинг
- [ ] Документация
- [ ] Тесты

## Чеклист
- [ ] Код следует стандартам проекта
- [ ] Добавлены тесты
- [ ] Обновлена документация
- [ ] Все тесты проходят
- [ ] Добавлена запись в CHANGELOG.md

## Скриншоты (если применимо)
...

## Related Issues
Closes #123
```

### Ревью кода

- Минимум 1 аппрув от мейнтейнера
- Все комментарии должны бытьresolved
- CI/CD пайплайны должны проходить успешно

## 🐛 Сообщение об ошибках

### Перед созданием issue

1. Проверьте существующие issues
2. Обновитесь до последней версии
3. Попробуйте воспроизвести проблему

### Шаблон bug report

```markdown
**Описание проблемы**
Чёткое описание проблемы.

**Шаги воспроизведения**
1. Запустить команду '...'
2. Отправить сообщение '...'
3. Увидеть ошибку

**Ожидаемое поведение**
Что должно было произойти.

**Скриншоты**
Если применимо.

**Окружение:**
- OS: [например, Ubuntu 22.04]
- Python: [например, 3.11]
- Версия проекта: [например, 2.0.0]

**Логи**
```
Вставьте логи здесь
```

**Дополнительно**
Любая дополнительная информация.
```
